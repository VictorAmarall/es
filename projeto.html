Projetar Modelo de Classe

Projeto

Finalidade

É desenvolver um determinado sistema de software segundo uma metodologia de engenharia de Software.

Observações

Um processo de desenvolvimento corresponde a um conjunto de atividades a serem desenvolvidas por um ou mais trabalhadores da indústria do software.

Artefatos de Entrada

  • DCC Diagrama de Classes Conceituais
  • LA Lista de Atributos
  • EDR Especificação dos Relacionamentos

Artefatos de Saída

DCU Diagrama de classe unificados

Papel Responsável:

Analista de Sistema

Ferramentas

  • Enterprise Architect
  • StarUML
  • Visual Paradigm for UML

Passos

Convergir as diversas outras linguagens de modelagem utilizadas em diferentes processos de desenvolvimento de software

Passos Alternativos

Sumarizar alguns dos diagramas utilizados na linguagem UML.

Levantar Arquitetura

Projeto

Finalidade

Projetar detalhadamente a estrutura e o comportamento interno de cada subsistema (módulos) através dos artefatos de entrada. Cada módulo tem um conjunto de detalhamento e processos onde apenas usuários específicos utilizaram esses subsistema que faz parte do projeto (Sistema completo)

Observações

Deve ser feito em camadas.

Artefatos de Entrada

  • DC Diagrama de Componentes
  • DE Diagrama de Estado
  • DO Diagrama de Objetos

Artefatos de Saída

DM Detalhamento do modulo ou subsistema

Papel Responsável:

Arquiteto

Ferramentas

  • BPM ferramenta para modelar
  • BPMN modelagem de negócios

Passos

  1. 1-Verificar os arquivos de entrada.
  2. 2-Dividir o projeto em módulos.

Passos Alternativos

Validar Interface

Projeto

Finalidade

Validar a interface criada pelo designer junto ao cliente.

Observações

A interface validade só passa pra próxima fase caso tenha sido aprovada pelo cliente.

Artefatos de Entrada

IV Interface Visual

Artefatos de Saída

DA Documentação de aprovação

Papel Responsável:

  • Cliente
  • Gerente de Projeto

Ferramentas

  • Photoshop
  • Microsoft Power Point

Passos

  1. Marca um evento junto ao cliente
  2. Apresentar interface proposta

Passos Alternativos

Enviar arquivos pelo email caso o cliente não tenha tempo e solicitar a aprovação ou reprovação da interface apresentada.

Projetar Protótipo Visual

Projeto

Finalidade

Mostrar o software que irá ser implementado de forma rápida e econômica.

Observações

Deve conter apenas arquivos visual nada implementado.

Artefatos de Entrada

DM Detalhamento do Modulo ou Subsistema

Artefatos de Saída

IV Interface Visual

Papel Responsável:

Designer

Ferramentas

  • Balsamiq
  • MockFlow

Passos

  1. Analisar o detalhamento do modulo.
  2. Criar o projeto visual.

Passos Alternativos

A programação deve seguir o visual e não o visual seguir a programação.

Elaborar Diagramas UML

Projeto

Finalidade

É desenvolver diagrama de objetos, diagrama de estados e diagrama de componentes através dos dados de entrada para auxiliar a divisão do projeto.

Observações

Cada diagrama deve conter todas as características da UML.

Artefatos de Entrada

  • DCC Diagrama de Classes Conceituais
  • LA Lista de Atributos
  • EDR Especificação dos Relacionamentos

Artefatos de Saída

  • DC Diagrama de Componentes
  • DE Diagrama de Estado
  • DO Diagrama de Objetos

Papel Responsável:

Analista de Sistema

Ferramentas

  • Enterprise Architect
  • StarUML
  • Visual Paradigm for UML

Passos

  1. Detalhar e criar o diagrama de objetos.
  2. Detalhar e criar o diagrama de componentes.
  3. Detalhar e criar o diagrama de estado.

Passos Alternativos

Utilizar outra ferramenta para criar os diagramas.
