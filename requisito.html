Elaborar Método

Levantamento de Requisito

Finalidade

Planejar métodos que sirvam para a realização de etapas na fase de requisitos.

Observações

É necessário que seja feito um cronograma para as realizações das atividades.

Artefatos de Entrada

SG Sugestão Comercial

Artefatos de Saída

OEL Organização das Etapas do Levantamento

Papel Responsável:

Gerente de Projeto

Ferramentas

  • Microsoft Word
  • Microsoft Excel

Passos

  • 1.Relatar todo o processo da entrevista inicial.
  • 2.Criar cronograma.
  • 3.Descrever os métodos de cada etapa.

    • Passos Alternativos

    Se no 1° passo não houver clareza no processo relatado, consequentemente o 2° passo será afetado. Caso isso ocorra é necessário que seja realizado outra entrevista apenas para corrigir o entendimento

    Definir Requisitos

    Levantamento de Requisito

    Finalidade

    Definir um conjunto de requisitos a serem extraídas do processo e todas as decisões relacionadas aos requisitos sejam embasadas em dados reais relatados.

    Observações

    O Analista de Requisitos deve está atento aos requisitos para que evite a reprovação e seja realizada o processo novamente.

    Validar Documentação

    Levantamento de Requisito

    Finalidade

    Visar a assegurar que o software corresponda aos requisitos levantados.

    Observações

    O objetivo da validação de requisitos é descobrir erros nos requisitos documentados.Só deve seguir para a próxima fase se todos os requisitos forem aprovados.
